title: ctx.tools.llm

---

#

## Shape

```ts
interface LLMTools {
  call(input: string | Message[]): Promise<Message>
}
```

## Overview

`ctx.tools.llm` allows a strategy to call the currently selected model directly.

This method performs a **single model call** and returns the assistant message.

It is intended for **internal strategy tasks**, such as:

- summarizing history
- rewriting prompts
- extracting information
- classifying intent
- compressing context

The model configuration (provider, temperature, output limits, etc.) is taken from the user's active model settings.

Strategies do not control model parameters directly.

Calls made through ctx.tools.llm.call() do not automatically modify the conversation history.
Strategies may choose to store the result manually if needed.

This call is independent from ctx.slots.
It does not affect the prompt being constructed for the current turn unless the strategy explicitly inserts the result into a slot.

Calls made through this API still consume model tokens and may count toward the user’s model usage depending on the runtime configuration.

## call(input)

Calls the active model using the provided input and returns the assistant message.

### Parameters

#### input

The input sent to the model.

**Type:** `string | Message[]`

If a string is provided, it is automatically converted into a single user message:

```
"Hello"
```

becomes

```
[{ role: "user", content: "Hello" }]
```

If a `Message[]` is provided, the messages are sent to the model as-is.

Example using a string:

```
await ctx.tools.llm.call("Summarize the following text...")
```

Example using messages:

```
await ctx.tools.llm.call([
  { role: "system", content: "Summarize the following conversation." },
  { role: "user", content: conversationText }
])
```

### Return Value

Returns the assistant message generated by the model.

**Type:** `Promise<Message>`

Example returned object:

```
{
  role: "assistant",
  content: "The conversation is mainly about..."
}
```

## Usage

Example: summarizing conversation history before storing it.

```ts
export async function onTurnEnd(ctx) {

  const recent = ctx.history.recent(20)

  const summary = await ctx.tools.llm.call([
    {
      role: "system",
      content: "Summarize the following conversation."
    },
    {
      role: "user",
      content: JSON.stringify(recent)
    }
  ])

  ctx.memory.set("history_summary", summary.content)
}
```

Example: quick classification task.

```
const result = await ctx.tools.llm.call(
  `Classify the intent of this message: ${ctx.input}`
)
```

## Notes

`ctx.tools.llm.call` performs a **single model invocation**.

It is typically used for internal processing tasks inside a strategy.

The model configuration is determined by the user's active model settings rather than by the strategy itself.
