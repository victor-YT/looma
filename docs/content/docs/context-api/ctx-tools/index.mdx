# ctx.tools

`ctx.tools` is the host-backed operations surface available inside strategy workers.

## Current Tool Groups

The live runtime currently exposes:

- `ctx.tools.llm`
- `ctx.tools.state`
- `ctx.tools.memory`

## `ctx.tools.llm`

This includes:

- `call(messages, options?)`
- `run({ messages, tools, toolChoice, maxRounds, temperature })`

These methods execute through the host. They do not call provider SDKs directly inside the worker.

## `ctx.tools.state`

This includes:

- `get(key)`
- `set(key, value)`
- `delete(key)`

State is persisted through host-side strategy state storage rather than worker memory alone.

## `ctx.tools.memory`

This includes:

- `query(options?)`
- `search(query, options?)`
- `ingest(input, options?)`
- `readAsset(assetId, maxChars?)`
- `retireBySourceMessage(messageId)`
- `retireMemory(memoryId)`

This is the main strategy-facing Memory Cloud and memory-ingest surface today.

## Tool Execution Hook

The current runtime also supports a strategy `onToolCall(ctx, call)` hook.

If a strategy does not define it, host-side tool execution is used as the default path.
